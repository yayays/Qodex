# Qodex v0.1.3

## Highlights

- Improved OpenCode compatibility by preferring the global event stream when available
- Surfaced nested OpenCode session errors so upstream API failures are no longer masked by a generic fallback
- Added `permission.asked` handling and plural permission-response fallback support for newer OpenCode servers
- Added managed background host and restart flows with PID-backed process tracking and `ps` fallback
- Added `/restart` support in the hosted runtime and aligned the CLI, quick-start, and README command documentation

## Included

- OpenCode backend fixes for event handling, error extraction, permission approval flow, and todo-event noise suppression
- `host:qodex:bg`, `start:qodex:bg`, `restart:qodex`, and `restart:qodex:bg` scripts for managed foreground/background operation
- PID, state, and log files under `/tmp` for managed host instances
- Restart completion notifications routed back to the requesting conversation
- CLI and runtime help updates for `/restart` and autocontinue-related command coverage
- README, Chinese README, AGENTS guide, and edge README updates for the unified start/restart workflow

## Notes

- `restart:qodex` now prefers managed runtime metadata in `/tmp` and falls back to process matching when metadata is unavailable
- Background start is intended for the managed host entrypoint rather than ad hoc shell backgrounding of the restart helper
- Release verification covered `cargo test -p qodex-core`, `npm --workspace @qodex/edge run check`, edge runtime tests, and restart-script tests
